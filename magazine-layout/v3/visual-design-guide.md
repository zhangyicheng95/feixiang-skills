# 高级视觉排版规则

## 目标

精美排版不是把原文档截图贴进 HTML，而是把已识别内容重新组织成“设计师排过”的纸面资料。

最终交付必须包含两层：

1. 重排正文：可复制、可编辑、可打印的 HTML 内容，是主交付。
2. 原卷校验附录：逐页图片或题内原图，只用于核对题量、图形和公式，不得替代正文。

如果当前链路无法 OCR 或结构化出完整题目，必须停止说明缺少 OCR/题目结构化结果，不能用整页图片冒充排版。

## 高级感来源

高级感来自信息秩序，不来自花哨装饰：

- 先声明设计系统：生成 HTML 前先在内部确定页面性格、字体、颜色、间距、线条、图形处理、题型模块规则；不要边写边随意加样式。
- 统一网格：稳定页边距、题区间距、题号宽度、答题区长度。
- 清晰层级：标题、考试信息、题型标题、题干、选项、答题区视觉层级分明。
- 高信息密度：充分利用 A4 纸面，但保留呼吸感。
- 克制配色：默认黑白灰加一枚低饱和强调色，避免大面积彩底。
- 精准对齐：题号、选项、横线、表格、分栏边界必须对齐。
- 可打印优先：白底、深色文字、细线框、浅灰底纹，不依赖屏幕特效。

## 纸面设计系统

每份试卷/讲义都必须像一个完整设计系统，而不是默认 CSS 模板。生成前先确定以下 token，并在 CSS 变量中落地：

```css
:root {
  --paper: #fffdf7;
  --ink: #1f2623;
  --muted: #65736e;
  --hairline: #d8ded8;
  --wash: #f4f6f2;
  --accent: #355c4a;
  --accent-soft: #e8f0ea;
  --radius-sm: 4px;
  --space-1: 2mm;
  --space-2: 4mm;
  --space-3: 6mm;
}
```

设计要求：

- 标题区要有“资料身份”：学段/学科/年份/地区/用途作为 eyebrow，主标题突出但不压迫正文。
- 题型区要有稳定节奏：题型标题、说明、题目列表、答题区之间有统一间距。
- 题号要成为视觉锚点：固定宽度或圆角徽标，避免题号和题干挤在一起。
- 选项要跟随内容长度选择 1/2/4 栏，不允许短选项散乱、长选项被压扁。
- 答题线使用真实 CSS 线条，不使用连续下划线字符；作文/解答题使用浅灰横线或网格。
- 页面装饰只服务信息层级：细线、浅底、留白、编号徽标即可；不要大面积渐变、贴纸、emoji、强阴影。
- 打印预览必须成立：去掉屏幕阴影后仍有清晰层级。

## 图片还原阶梯

试卷上的图很难“重绘准确”，所以默认策略是**真实图优先，重绘最后**：

1. **题内图裁剪 URL**：最优。后端/OCR 返回 `figure.url`，直接放回对应题目附近。
2. **题内图 bbox + 原页图 URL**：可接受。用后端裁剪服务或 CSS 裁剪生成真实原图局部；必须有明确页码和 bbox，不能凭感觉估位置。
3. **原页图 + 明确题号定位**：临时兜底。正文保留完整题干，并在题目附近嵌入原页局部/整页缩略图卡片；如果没有 bbox，只能作为校验，不得声称“题内图已还原”。
4. **可确定的简单矢量图**：只有当 OCR 明确给出图形几何参数、坐标、标签、数据时才允许用 SVG 重绘，例如“半径 6 cm 的圆并标注 r=6cm”。统计图、电路图、几何组合图默认不得猜画。
5. **缺图阻塞**：题目依赖图片且没有真实图片 URL/bbox/结构化图形数据时，停止生成，不交付 HTML。

禁止把“图形题”处理成纯文字摘要。准确性高于好看：宁可保留一张朴素的真实截图，也不要生成一张漂亮但数据错误的图。

## 题内图片版式

带图题必须把图片放在题目语义附近，不要只丢到最后附录。推荐结构：

```html
<li class="question question-with-figure keep-together">
  <div class="question-main">
    <div class="question-body">12. 如图，...</div>
    <div class="answer-area"></div>
  </div>
  <figure class="question-figure">
    <img src="题内真实图片URL" alt="第 12 题原图" />
    <figcaption>第 12 题图</figcaption>
  </figure>
</li>
```

推荐 CSS：

```css
.question-with-figure {
  display: grid;
  grid-template-columns: minmax(0, 1fr) 42mm;
  gap: var(--space-2);
  align-items: start;
}

.question-figure {
  margin: 0;
  padding: 2mm;
  border: 1px solid var(--hairline);
  background: #fff;
  break-inside: avoid;
}

.question-figure img {
  display: block;
  max-width: 100%;
  height: auto;
}

@media print {
  .question-with-figure {
    break-inside: avoid;
  }
}
```

如果图片较宽，回退为上下结构；不要强行压到右栏导致图中文字看不清。

```html
<main class="exam-layout">
```

## 视觉规范

- 页面：A4 纵向，`12mm 14mm` 到 `14mm 16mm` 页边距。
- 字体：中文优先 `Noto Serif SC`, `Songti SC`, `SimSun`；题干可用衬线，辅助信息可用无衬线。
- 主标题：22-28px，字重 700，避免过大。
- 题型标题：用细线、浅底或编号徽标区分，不使用大色块。
- 题目：题号固定宽度，题干和选项左对齐。
- 选项：数学选择题优先 2 栏或 4 栏网格，短选项不得散乱。
- 填空：使用真实边框线或 `border-bottom`，不能用一串下划线字符。
- 表格：细线边框、表头浅灰底，禁止重边框。
- 公式：MathJax 渲染，行内公式不破坏行高，展示公式保持居中。
- 图形：能提取题内原图时放回题目附近；不能提取时，如果有 bbox 或裁剪图就使用真实原图局部；如果只有整页附录图，不能把它当题内图还原，图形依赖题必须阻塞。
- 图形题：如果无法提取题内图，不能写“请参考原卷图示”来糊弄；要么回填真实题内图，要么停止说明缺少题内图片裁剪/定位能力。
- 统计图题：必须保留图表标题、数据、子问和判断项，不能简写成“统计图分析”。
- 重绘限制：除非题目结构中明确给出全部几何/统计/坐标/标签数据，否则不要用 SVG/Canvas 重画原图；优先使用真实图片裁剪。

## 禁止的低级排版

- 正文只有整页原卷图片。
- 只有图片，没有可编辑题干、题号、题型标题。
- 用“原卷保真层”代替重排正文。
- 用“请参考原卷图示”“统计图分析”“补充统计图”等摘要替代图形题正文。
- 把一道含多个子问的大题压成一句概括。
- 大面积渐变、黑底白字、花哨贴纸、无意义英文装饰。
- 题目卡片间距过大，导致 A4 纸张浪费。
- 每题都加厚重边框，视觉噪声过强。
- 题号、选项、横线不对齐。

## 验收

- [ ] 正文主体是可复制 HTML，不是整页图片。
- [ ] HTML 中有明确题型结构和题号结构。
- [ ] 原卷图片只在附录或题内图位置出现，不充当整页正文。
- [ ] 图形题、统计图题没有被摘要或“参考原卷”替代。
- [ ] 样式高级克制，适合打印和分发。
- [ ] 纸面信息密度合理，没有大面积空白或低幼装饰。
- [ ] HTML 使用统一设计 token，而不是零散硬编码颜色和间距。
- [ ] 带图题的图片出现在题目附近，且来源是真实 URL 或后端裁剪结果。
