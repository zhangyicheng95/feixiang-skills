# 公式、原图与题量保真规则

## 目标

试卷/题单类排版的最低标准是“像原卷一样完整可用”。美化不能牺牲三件事：

1. 分数、根式、方程等数学内容必须正确显示。
2. 原卷中的图形、表格、图片必须保留。
3. 题目数量不能缩水。

## 分数与公式渲染

### 必须识别的数学源码

输入中出现以下任一形式时，必须走数学渲染：

- `\(\dfrac{7}{9}\)`、`\(1\dfrac{2}{9}\)`
- `\frac{3}{8}`、`\dfrac{3}{11}`
- `27.2\%`、`\times100\%`
- 根式、方程、比例式、带单位公式

### HTML 输出规则

- 必须引入 MathJax。
- LaTeX 分隔符在 HTML 文本节点中使用单反斜杠：`\(...\)`，不要输出成 `\\(...\\)`。
- 不要把 LaTeX 放进被转义的字符串、`code`、`pre`、`textarea` 中。
- 不要把 `\dfrac` 改写成纯文本 `7/9`，小学数学试卷需要竖式分式显示。
- 如果模型从 JSON 字符串中拿到双反斜杠，写入 HTML 前要还原为单反斜杠。

### 推荐结构

```html
<span class="math">\( \dfrac{7}{9} \)</span>
<span class="math">\( 1\dfrac{2}{9} \)</span>
<span class="math">\( \dfrac{198}{198+102}\times100\% \)</span>
```

## MathJax 与 Paged.js 顺序

Paged.js 分页前必须先完成公式排版，否则分数可能以源码形式显示或导致分页错位。模板中使用：

```html
<script>
  window.MathJax = {
    tex: {
      inlineMath: [['\\(', '\\)'], ['$', '$']],
      displayMath: [['\\[', '\\]']]
    },
    svg: { fontCache: 'global' },
    startup: { typeset: false }
  };
</script>
<script src="https://cdn.jsdelivr.net/npm/mathjax@3/es5/tex-svg.js"></script>
<script>
  window.PagedConfig = {
    before: async () => {
      if (window.MathJax?.typesetPromise) {
        await window.MathJax.typesetPromise();
      }
    }
  };
</script>
<script src="https://unpkg.com/pagedjs@0.4.3/dist/paged.polyfill.js"></script>
```

## 原图保留

### 零占位原则

“保留原图”必须是 HTML 中真实出现原图资源，不是写一句说明。

禁止：

- `[此处保留原卷图形]`
- `[此处原卷为：...]`
- `[原图见附件]`
- `图略`
- `占位说明`
- 用纯文字描述替代几何图、统计图、表格图。

PDF/截图类试卷交付时，如果 HTML 中没有真实 `<img>`，即判定原图保留失败。`<object>`、`<embed>`、`<iframe>` 可能在浏览器或打印环境中显示为空白，不能作为原图保留通过依据。

### 必须保留的内容

- 几何图、对称轴图、长方形/圆/三角形图。
- 统计图、表格、坐标图。
- 电路图、光路图、实验装置图。
- 原卷中的插图、示意图、阅读配图。

### 处理规则

- 优先复用上传文档中已有图片资源。
- 如果能裁剪出题内图，就把图放回对应题目附近。
- 如果无法稳定裁剪或重绘，保留原页面截图作为 `figure`，再在旁边/下方重排文字。
- 如果无法拿到逐页截图或题内原图，即使能拿到原 PDF URL，也不能用 `<object>` 或 `<iframe>` 冒充保真；必须说明当前链路缺少 PDF 转图/图片提取能力。
- 不得按 resourceId 或文件名猜测图片地址；图片 URL 必须来自工具返回结果。
- 不得给图片添加 `onerror` 来隐藏加载失败，图片加载失败就表示保真失败。
- 如果没有工具返回的真实图片 URL，必须停止生成 HTML；不能在正文里写 `[此处原卷为...]` 来替代图片。
- 学科图不能“凭感觉重画”，尤其不能用 AI 生成一张相似但数据不同的图。
- 如果某题依赖图片但图片无法提取，必须在输出前标记为阻塞，而不是直接删掉该题。

### 图片 HTML 结构

```html
<figure class="source-figure keep-together">
  <img src="原图URL或上传资源URL" alt="原卷第2题图形" />
  <figcaption>原卷图形，已保留</figcaption>
</figure>
```

PDF 逐页图片结构：

```html
<section class="source-pages">
  <figure class="source-page keep-together">
    <img src="原卷第1页图片URL" alt="原卷第 1 页" />
  </figure>
</section>
```

禁止把 PDF URL 写进 `<object>`、`<embed>`、`<iframe>` 后声称“原卷保真层”。如果打印预览里是空白框，就是失败。

禁止编造逐页图片地址，例如 `..._page_1.png` 或 `...pdf?page=1`。这些地址即使看起来合理，只要不是工具返回的真实 URL，都是伪保真。

## 题量不缩水

### 计数门槛

生成 HTML 前后必须做计数：

```plain
原始题量：
一、选择题 10 题
二、填空题 5 题
三、解答题 4 题

HTML 题量：
一、选择题 10 题
二、填空题 5 题
三、解答题 4 题
```

两个计数不一致时，不能交付。

### 排版策略

- 一页放不下就自动分页，不删题。
- 图片太多就增加页数，不省略图片。
- 表格太宽就缩小字体或横向滚回单栏，不删列。
- 题干太长就回到单栏，不压缩成不可读的小字。
- 禁止用“更多题目请参考原卷”“请参考原卷保真层”代替未重排的题目。

## 自检清单

- [ ] 页面里没有裸露的 `\(\dfrac`、`\frac`、`\times` 源码。
- [ ] 所有数学表达式都在浏览器中显示为公式。
- [ ] 原卷中每张图都在 HTML 中有对应图片或原页面截图。
- [ ] 没有“此处保留”“此处原卷为”“图略”“占位”等伪保真文字。
- [ ] 没有“请参考原卷保真层”“更多题目请参考原卷”等偷懒话术。
- [ ] PDF/截图类交付中存在真实 `<img>`，不是空白 `<object>`、`<embed>` 或 `<iframe>`。
- [ ] 所有图片地址来自工具返回结果，没有猜测拼接。
- [ ] 没有 `onerror` 隐藏图片加载失败。
- [ ] 没有真实图片 URL 时，已停止生成 HTML，而不是用文字描述替代图片。
- [ ] 图形依赖题没有因为图片缺失而被删除。
- [ ] 原始题量与 HTML 题量一致。
- [ ] 因为内容多产生多页是正确结果，不允许为压缩页数删内容。
