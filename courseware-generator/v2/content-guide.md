# 课件内容生成指南

---

## 一、工作流总览

```
步骤 1  初始化：调用 continue_ask 向用户确认课件偏好参数（⏱ 倒计时自动确认）
  ↓
步骤 2  检索：根据确认的参数，搜索原文 / 教案 / 教学资料
  ↓
步骤 3  规划：确定教学目标 → 设计课件结构 → 逐页大纲（含内容 + 交互）
  ↓
步骤 4  生成：按大纲分批生成各页具体内容
  ↓
步骤 5  验收：逐页核对大纲 → 组装 → 交付
```

> **⚠️ 核心原则：趁用户在场先确认，确认不阻塞。** 用户发完需求后注意力最集中，立即调用 `continue_ask` 确认偏好参数（带倒计时，超时自动确认）；确认后再检索资料、规划大纲、分批生成。

---

## 二、步骤 1 —— 初始化课件项目（调用 `continue_ask`）

**⚠️ 硬性要求：步骤 1 只做一件事——调用 `continue_ask` 向用户确认参数。禁止在本步骤中同时调用搜索工具、开始规划或执行任何其他操作。必须等 `continue_ask` 返回结果（用户确认或倒计时自动确认）后，才能进入步骤 2 检索。**

用户发出课件需求后，**立即调用 `continue_ask` 确认偏好参数**，趁用户还在界面上。

### 1.1 参数列表

| 参数 | 是否必填 | 说明 | 来源 |
|------|----------|------|------|
| **课题** | 必填 | 课件的主题 / 课文名 / 知识点 | 用户输入 |
| **年级** | 必填 | 学段 + 年级 | 用户输入 或 从课题推断 |
| **教材版本** | 必填 | 人教版 / 北师大版 / 苏教版等 | 用户输入 或 默认「人教版」 |
| **课型** | 必填 | 新授课 / 复习课 / 练习课 | 用户输入 或 默认「新授课」 |
| **页数** | 必填 | 课件总页数（含封面与结束页） | 用户输入 或 默认 10 页 |
| **教学重点** | 选填 | 用户希望重点讲解的内容 | 用户输入 或 留空（后续检索后由 AI 确定） |

### 1.2 执行规则

1. **整理已知信息**：从用户输入中提取已提供的参数。
2. **补全缺失项**：对用户未指定的参数，根据课题名称做基本推断或使用合理默认值。
3. **只调用 `continue_ask`，不调用任何其他工具**：本步骤禁止同时调用搜索工具、文件操作或任何其他工具。因为检索必须基于用户确认后的参数（课题+年级+教材版本）才有意义，在确认之前检索可能搜错方向。

### 1.3 调用 `continue_ask` 工具（⏱ 带倒计时）

**必须调用 `continue_ask` 工具**向用户呈现参数并请求确认，**不能仅用普通文本消息代替**。

**确认清单内容示例**：

```
⏱ 课件信息确认（倒计时结束后自动确认）

• 课题：《少年闰土》
• 年级：小学六年级
• 教材版本：人教版（部编版）
• 课型：新授课
• 页数：10 页
• 教学重点：（可填写，如"人物描写手法"，不填则由 AI 根据教案确定）

如需调整，请直接在聊天里回复您的想法。我会以您的说明来制作课件～
```

### 1.4 倒计时与自动确认机制

- `continue_ask` 呈现后启动**倒计时**。
- **用户在倒计时内回复**：按用户修改后的参数继续。
- **倒计时结束用户未回复**：以当前参数（用户已提供 + 推断 + 默认值）**自动确认**，直接进入步骤 2。
- **禁止无限等待**：不得因为用户未回复就停滞不前。倒计时到期即视为用户认可当前参数。

## 三、步骤 2 —— 检索原文与教学资料

**⚠️ 硬性要求：不得在缺乏资料的情况下做任何课件内容规划。必须先完成检索，再进入步骤 3 规划。**

### 2.1 须检索的内容

| 检索类型 | 搜索关键词示例 | 目的 |
|----------|----------------|------|
| **原文 / 原始材料** | 「少年闰土 课文原文」「分数乘法 定义与公式」 | 获取准确的课文文本、知识点定义、公式等一手内容，确保课件引用准确、不编造。 |
| **教案 / 教学设计** | 「少年闰土 六年级 教案」「分数乘法 小学 教学设计」 | 了解该课题的教学目标、环节划分、课时安排、重难点，为步骤 3 的结构规划提供依据。 |
| **补充资料**（可选） | 「少年闰土 写作背景 鲁迅」「分数乘法 常见易错题」 | 丰富课件深度：背景知识、拓展素材、典型错题等。 |

### 2.2 检索要求

- 至少完成**原文 + 教案**两类检索，各取 1～2 份有效结果阅读。
- 阅读后须提炼出以下信息，供步骤 3 规划使用：
  - 课文/知识点的**核心内容摘要**（原文关键段落、公式、定义等）；
  - 教案中的**教学目标、环节划分、课时安排、重难点**；
  - 可用的**补充素材**（背景故事、拓展知识、易错点等）。
- 若步骤 1 中「教学重点」留空，此时根据检索到的教案重难点确定。

## 四、步骤 3 —— 规划课件结构与逐页大纲

基于步骤 1 确认的偏好参数和步骤 2 的检索资料，依次完成以下子步骤。

### 3.1 确定教学目标

- **学习目标**：1～3 条，尽量**可观测**（如「能正确朗读课文并概括闰土的四件事」），避免仅用「了解」「理解」。
- **重点与难点**：各 1～2 条。
- 目标须与用户在步骤 1 中确认的**教学重点、课型**对齐，同时参考步骤 2 检索到的教案目标。

### 3.2 设计整体结构

用一句话概括课件叙事线，标明每段的**教学功能**与**页数分配**：

> 示例：封面(1p) → 学习目标(1p) → 课文导入：背景介绍(1p) → 课文精读：四件事(3p) → 人物形象分析(1p) → 写作手法(1p) → 课堂练习(1p) → 小结(1p)

- 总页数须与步骤 1 确认的页数一致。
- 页数分配须服务于教学目标，**不得先定页数再凑内容**。

### 3.3 逐页大纲（核心输出）

**每页须按以下模板填写**，后续生成与验收时逐项核对：

| 字段 | 说明 |
|------|------|
| **页码** | 第几页 |
| **页标题** | 如「课文精读——闰土的四件事」 |
| **页类型** | 封面 / 目录 / 讲解 / 精读 / 分析 / 例题 / 练习 / 讨论 / 小结 / 结束 |
| **服务的学习目标** | 本页对应哪条目标（标序号） |
| **必须出现的具体内容** | **逐条列出**，具体到可核对。如：① 课文第3段原文引用 ② 「雪地捕鸟」事件概括1句 ③ 闰土动作描写摘录2处 ④ 对应的人物特点归纳1句 |
| **交互设计** | 本页的互动功能。如：点击显示答案 / 选择题点选判对错 / 拖拽排序 / 填空输入 / 动画逐步展示 / 无交互（纯展示）。**禁止设计依赖方向键（↑↓←→）或空格键的交互**，这些按键已被壳框架占用于翻页，会产生冲突。 |
| **与前后页的衔接** | 如「上一页导入背景，本页进入课文精读；下一页继续第二件事」 |

**规划原则**：
- 内容必须基于步骤 2 检索到的**真实原文和资料**，禁止凭空编造课文内容或知识点。
- 每页内容量须匹配 16:9 单屏画布，**不过满也不过空**。内容多则拆页，少则合并或补充。
- 含交互的页须注明「须支持完成态」（用于倒序翻页时直接展示结果）。
- **禁止使用方向键（↑↓←→）和空格键作为交互手段**：壳框架在放映模式下会无条件拦截这些按键用于翻页，页面内的方向键/空格键监听将不会生效。交互应基于**点击、触摸、拖拽**等不与翻页冲突的方式。

规划完成后**直接进入步骤 4 生成**，无需等待用户确认大纲。

## 五、步骤 4 —— 分批生成各页内容

### 5.1 核心原则：先分片生成，最后一次性组装

> **⚠️ 严禁「边生成边往 HTML 文件里插入」。** 课件 HTML 文件很大（十几页 content 字符串），对已有文件做字符串匹配编辑（str_replace / multi_edit）极易因空白符、转义符差异而匹配失败，导致不得不重写整个文件并丢失已生成内容。
>
> **正确做法**：每批只生成独立的**页面片段文件**，全部批次完成并验收后，在步骤 5 中**一次性组装**为最终 HTML 文件。**在步骤 4 期间不创建、不编辑任何 .html 课件文件。**

### 5.2 分批策略

- **不一次性生成全部页**。推荐每批 2～4 页，复杂页（含交互、大量内容）单独一批。
- 推荐分批顺序：封面+目录 → 核心讲解页 → 练习/互动页 → 小结/结束页。

### 5.3 每批的输出格式——独立的页面片段文件

每批生成后，将结果保存为**独立的 HTML 片段文件**（如 `pages-batch1.html`、`pages-batch2.html`），文件内容**只包含本批各页的 `<template>` 标签**，不包含 HTML 壳、不包含其他批次的页面。

单个片段文件格式示例（`pages-batch1.html`，包含第 1～3 页）：

```html
<!-- 第 1 批：第 1–3 页 -->
<template class="page-data" data-id="1" data-name="封面">
  <style>
    body { display:flex; align-items:center; justify-content:center; background:linear-gradient(135deg,#667eea,#764ba2); color:#fff; }
    h1 { font-size:42px; }
  </style>
  <div><h1>少年闰土</h1><p>部编版 · 小学语文六年级上册</p></div>
</template>

<template class="page-data" data-id="2" data-name="目录">
  <style>/* ... */</style>
  <!-- 第2页内容 -->
</template>

<template class="page-data" data-id="3" data-name="作者与背景">
  <style>/* ... */</style>
  <!-- 第3页内容 -->
</template>
```

**关键规则**：
- 每个片段文件只含若干 `<template class="page-data">` 标签，不含 `<!DOCTYPE>`、`<html>`、`<head>`、`<body>` 等外层结构。
- `<template>` 内**不要写** `<!DOCTYPE>`/`<html>`/`<head>`/`<body>`——壳框架会自动包装。
- `<template>` 内的 `<script>` 中不可出现字面量 `</template>`，如需要用 `<\/template>` 替代。
- 文件名建议包含批次号和页码范围，便于后续识别：`pages-batch1-p1to3.html`。

### 5.4 每批生成时须携带的上下文

传给实现者（或 sub-agent）的输入**不能只有本页大纲**，须包含完整的课件语境：

| 必传信息 | 说明 |
|----------|------|
| 课件基本信息 | 课题、学科、年级、教材版本、配色风格 |
| 教学目标 | 步骤 3.1 的 1～3 条学习目标 |
| 本批各页的完整大纲 | 步骤 3.3 模板的所有字段 |
| 本批在整体中的位置 | 第几页到第几页、前一页标题、后一页标题 |
| 相关原文/素材 | 本批页面需要用到的课文原文段落、知识点定义、背景资料等（从步骤 2 检索结果中摘取） |
| **输出格式要求** | 明确告知：只输出本批各页的 `<template>` 标签片段，不要生成完整 HTML 文件 |

### 5.5 生成规则

- **页数一致**：总页数必须与步骤 3 规划完全一致，不得多页、少页、合并或拆页。
- **内容一致**：每页内容须与大纲中的「必须出现的具体内容」逐条对应，不得遗漏、替换或大幅偏离。
- **引用准确**：课文原文、公式、定义等须与步骤 2 检索到的原始材料一致，禁止篡改或凭空编写。
- **交互实现**：大纲中标注了交互设计的页，须实现对应功能，并支持状态保存与恢复（`saveState` 上报 + `restoreState` 监听，供往回翻页时恢复用户离开时的状态）。
- **内容充实**：须按大纲填充实质内容，禁止只做版式骨架不填文字。
- **不碰 HTML 文件**：本步骤只产出片段文件，**禁止创建或编辑最终 .html 课件文件**。

## 六、步骤 5 —— 验收、组装与交付

### 6.1 逐页验收

全部批次生成完毕后（即所有片段文件都已产出），**逐页核对**：

| 验收项 | 标准 |
|--------|------|
| 页数一致性 | 片段文件中的总 `<template>` 数 = 步骤 3 规划的页数 |
| 内容一致性 | 每页实际内容与大纲「必须出现的具体内容」逐条对应 |
| 引用准确性 | 原文引用、定义、公式等与检索资料一致 |
| 交互完整性 | 标注了交互的页已实现 saveState 上报和 restoreState 监听，恢复后 UI 与离开时一致 |
| 内容充实度 | 讲解页有要点 ≥2 + 例证 ≥1；练习页有题干+答案；无「仅标题+一句话」的空壳页 |
| 无凑页 | 不存在两页讲同一件事且无递进 |
| data-id 连续性 | 所有页的 data-id 从 1 开始连续递增，无重复无跳号 |

- 不通过项须指明页码与缺失项，**退回对应批次的片段文件补全**（只重新生成该片段文件，不影响其他批次）。
- 全部通过后进入组装。

### 6.2 一次性组装（关键步骤）

> **⚠️ 这是唯一创建 .html 课件文件的时机。用一次 write/create 操作生成完整文件，不做后续编辑。**

组装步骤：

1. **按 data-id 顺序**读取所有片段文件（batch1 → batch2 → …），将各页 `<template>` 按序排列。
2. 将所有 `<template>` 标签套入 HTML 模板，**一次性写入**最终 .html 文件（HTML 模板结构详见 style-guide.md）。
3. **最终检查**：确认 `<template>` 数量与规划一致，每页 `data-name` 与大纲标题对应，`data-id` 连续。
4. 清理片段文件（可选）。
5. 交付时说明「直接浏览器打开即可，无需其他文件或服务器」。

### 6.3 如果组装后需要修改个别页

若交付后用户要求修改某几页：
- **不要**用 str_replace 在 HTML 文件中匹配和替换页面内容（大段 HTML 难以精确匹配）。
- **正确做法**：修改对应的片段文件 → 重新执行 6.2 的一次性组装 → 覆盖写入 .html 文件。即**任何修改都走「改片段 → 重新组装」的路径**。

## 七、分页类型最低内容标准

规划与生成时须对照，每页不得低于最低要求：

| 页类型 | 最低内容要求 |
|--------|-------------|
| 封面 | 标题 + 副标题（学科/年级/教材版本） |
| 目录 | 各节标题，可含页码或图标 |
| 讲解/精读 | 核心内容（原文/定义/规则）+ 要点 2～4 条或例证 ≥1；**禁止**仅「标题+一句笼统话」 |
| 分析/讨论 | 分析框架或讨论问题 + 要点/示例 ≥2 |
| 例题 | 题干 + 完整解答过程 + 可选思路/易错点 |
| 练习 | 题干 + 答案（或选项）；若有空间须有解析或提示 |
| 小结/结束 | 归纳或回顾要点 ≥2 条，**禁止**空白 |
