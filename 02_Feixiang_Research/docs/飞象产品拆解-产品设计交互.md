## 0. 一句话定位

> **飞象老师 = 教师用一句话（或上传一份资料）描述教学需求 → 多阶段 AI agent / workflow 编排 → 产出自包含、可交互的 HTML 教学课件 / 试题 / 教案的 AI 教学内容生成平台。**
>
> 它是一个**「开环生成器」**：只埋点停留时长，**不回收学生作答数据**。生成完即交付独立 HTML，"生成"与"使用"是两个孤岛。【实证（缺失证据）+ 推断】

产品对教师的核心承诺写在资源广场顶部 Banner 上：**「飞象老师，一句话生成专业互动课件」**。【实证，截图 05】

### 2.1 「置顶任务」暴露的能力线索【实证】

置顶任务区的命名直接暴露了产品的两类底层生成机制（**Agent 自主** vs **工作流**），对理解产品很关键：

| 置顶任务名 | 暴露的机制 |
|---|---|
| 教学动画 - **Agent自主规划** | Agent 自主决定步骤 |
| 练习数据回收 - **Agent自主调用** | Agent 自主调用工具 |
| 互动游戏 - **Agent自主调用** | Agent 自主调用工具 |
| 文档生成 - **工作流生成** | 固定 workflow 流水线 |
| 多页互动课件 - **工作流生成** | 固定 workflow 流水线 |

> 解读：飞象内部**同时存在两种生成范式**——"Agent 自主规划/调用"（灵活）与"工作流生成"（可控）。复杂多页课件走工作流，灵活任务走 Agent。详见技术篇 §2。

## 3. 逐功能模块拆解

### 3.1 AI 命题（featureId=17）【实证，截图 01/02】

- **用途**：根据素材（文章 / 上传文档 / 主题）命制试题。
- **入口**：左侧栏「AI命题」；或对话页顶部 Tab。
- **页面结构**：
  - 居中欢迎语「🐘 Hi, 我是飞象老师」；
  - 顶部三 Tab：**[AI命题] · AI组题 · AI教案·大单元**（当前选中带绿色下划线）；
  - 多模态输入框，占位符「请输入内容」，底部左侧附件📎/图片🖼图标，右侧语音🎙 + 圆形发送按钮（灰→可用时变绿）；
  - 「大家都在用」高频提示词列表，每条带 **🔥 热门角标 + 热度数字**。
- **示例提示词（热度，实证文本）**：
  1. 「根据我上传的 ChinaDaily 这篇文章，帮我命制北京英语中考阅读理解模拟题」 — 29672 热度
  2. 「请你找一篇适合中学生读的冰心散文，出一道中考记叙文阅读」 — 28508 热度
  3. 「根据人民日报"人工智能"主题报道，出一道北京西城区六年级难度应用题」 — 27168 热度
  4. 「结合古代科技相关情境，出一道"密度相关"中考难度的物理计算题」 — 26474 热度
- **关键交互**：点提示词条 → 自动填充输入框（推断）；提示词刻意包含**年级/科目/题型/难度/素材来源**多维度，是对教师的"输入范式"示范。
- **与其它模块关系**：与组题/教案同属一个对话壳，共享侧栏与输入区，仅示例与 featureId 不同。

### 3.2 AI 组题（featureId=18）【实证，截图 03/06】

- **用途**：组卷 / 题集编排，强调"分层、梯度、成套"。
- **示例提示词（文字含识别噪声）**：
  1. 「结合过往三年北京中考题和模拟题中考查"二次函数"的真题，组一份有难度梯度…」 — 29338 热度
  2. 「根据图中这名同学的错题，为他进行错因分析并组一套针对性练习」 — 28083 热度
  3. 「结合北上广深近三年中考语文约文言文阅读题，组一份题单，至少 30 题」 — 26044 热度
  4. 「我是广州一名初中英语老师，请帮我找过去五年广州中考英语真题」 — 29163 热度
- **差异点**：相对命题（出单题），组题强调"组一份/一套/题单"的**成套输出**与**分层难度**。

### 3.3 AI 教案·大单元（featureId=19）【实证，截图 04】

- **用途**：生成"大单元"教学设计 / 教案，对话页默认进入的 Tab（续段遍历从 featureId=19 落地）。
- **示例提示词（文字含识别噪声）**：
  1. 「帮我出一份课时教学设计，关于人教版新一上第四单元 11~20 的认识第 1 课时…」 — 29589 热度
  2. 「基于这份教案进行修改，在"0的意义"的部分变更为游戏化教学」 — 29471 热度（带 🔥 热门）
  3. 「结合"我校的玉兰树春天开花"这一情境，帮我生成语文部编版七上第一单元的…」 — 27236 热度
  4. 「基于这份教案进行修改，不改变其他内容，在一个环节中增加使用 AI 工具…」 — 26277 热度
- **差异点**：示例普遍是"**基于已有教案改写**"，呼应历史任务里大量「改编《…》/修改教案…」——说明**改编/迭代**是高频真实用法。

### 3.4 AI 互动课件 / 资源广场（featureId=20）【实证，截图 05/07】

- **用途**：① 生成互动课件的入口；② 同时是全站资源广场（浏览、检索、复用他人作品）。
- **页面结构**：
  - 顶部 Slogan「飞象老师，一句话生成专业互动课件」；
  - 多模态输入框，示例占位会动态轮播（实勘见两条）：
    - 「互动课件 | 帮我生成初среди英人教七下 U1 Grammar Focus 的互动课件，通过精简而高效的语言活动，帮助学生掌握并巩固与动物相关的词汇、句型及原因状语从句引导词 because 的用法。」
    - 「教学游戏 | 以《西游记》取经之路为故事线，生成六年级语文古诗词闯关游戏，让学生在游戏化体验中变"被动背诵"为"主动探究"。」
  - 输入框底部 4 个能力快捷按钮：**🎬教学动画 | 🎮教学游戏 | 互动课件 | 数据回收**；
  - 学科分类横向 Tab（12 类，数学带"热"角标）+ 右上「快速搜索资源」框；
  - **资源卡片瀑布流**（每行 4 列）。
- **资源卡片元素（实证）**：封面图（多为绿/蓝/黄主题插画或截图）+ 左上角角标（🔥热门 / 新手）+ 标题 + 作者头像与昵称 + ⭐/👁 浏览量。

  - 「【教学动画】生成语数英通用工具教学动画…」 — TeacherYO — 6127
  - 「【互动课件】小学数学 人教版六年级 圆柱的体积」 — 小时老师用AI — 5739

### 3.5 核心创作流（对话生成流程）【实证，来自历史完整交互日志】

> 实测案例：上传 71.5KB 九年级物理复习 Word → 输出 20 页课件，全程 8+ 阶段、2 个用户确认门控、9 个 Batch 串行生成。

**端到端阶段（产品视角）**：

```
① 信息确认表单（AI 预填默认值）
     ├ 课时内容（单选 + 自定义）
     ├ 年级刻 / 册次刻 / 教材版本刻（单选组，"刻"是飞象自造术语后缀）
     ├ 教学重点刻（多选，默认全选）
     ├ 课时数刻（1/2/3 课时以上）
     └ 特殊要求（自由文本）
     →【用户确认门控①】点"确认/不修改继续"，出现"已确认"标签
②  AI 专家思考（自动执行，可折叠卡片）
     🔍关键词提取 → 📖研读课程标准 → 📚教材内容检索（无资料时橙色告警）
     → 📖精选学科资源（外链 PDF）→ 🔍目标与重难点刻 → 🔍逐页内容设计（表格）
     →【用户确认门控②】审阅大纲、补充需求
③  课件生成计划（列出 4 个强互动页 + 16 普通页 + 9 个 Batch 划分）
④  分批生成进度（9 条进度行，逐条 🔄 → ✓）
⑤  交付物卡片（绿色 HTML 图标 + 文件名 + 缩略图预览 + ↓下载）
⑥  AI 说明文字（功能亮点 + 操作指南）
⑦  追问修复闭环（用户绿色追问卡片 → 定向修复 → 新版交付物卡片 + 修复摘要）
```

- **强互动页规格示例（专家思考输出，实证）**：
  - 第6页「热机循环」：4 个冲程按钮（吸气/压缩/做功/排气）+ 汽缸活塞同步动画 + 完成循环显示能量转化（Canvas + anime.js）。
  - 第11页「串并联电路连线」：左侧工具栏（电池/开关/灯泡）→ 拖导线连接 → 通路点亮、短路告警 → 获"电路工程师"勋章（SVG 拖拽 + 电路判定）。
- **追问修复实证**：用户追问「修复所有公式没有正常渲染的问题」→ AI 输出修复摘要 3 条（规范 `\( \)`/`\[ \]` 定界符、注入 MathJax 配置、公式排版优化）。

## 4. 信息架构与导航设计

- **三级导航体系**【实证】：
  1. **左侧栏（功能工具 + 任务管理）**：固定约 240px，白底，含 CTA、4 个 AI 工具、置顶/今日/历史任务、底部积分与用户名。

- **featureId 映射表（已确认）**：`17=AI命题`、`18=AI组题`、`19=AI教案·大单元`、`20=AI互动课件/资源广场`。
- **任务的持久化与可回溯**：历史任务以"标题"为索引常驻左栏，点击可回到该对话继续编辑/追问——**对话即文档**，每个生成会话是可回访的实体。

## 5. 设计系统

> 注意区分两层：**主站设计系统**（实勘所见，本节 5.1–5.6）与**课件产物内部的"设计令牌"**（每份课件各异，详见技术篇 §6 跨样本结论）。两者不是同一套东西。

### 5.1 色彩【实证 + 推断】

### 5.2 字体【实证】

- 字体族：`PingFang`（苹方，系统默认中文字体），基础字号 **14px**，行高 normal。
- 主标题（"Hi, 我是飞象老师"）显著加大加粗。

### 5.3 圆角 / 阴影 / 间距风格【实证 + 推断】

- 主站整体走**扁平、轻拟物**：欢迎区 textarea、广场卡片有**中等圆角 + 轻投影**（截图可见卡片悬浮感）；
- `getComputedStyle` 对 body/button 取到的 `border-radius:0 / box-shadow:none` 是兜底元素值，**不代表卡片组件**（卡片实测有圆角阴影，以截图为准）。
- 间距推断 8px 基线（8/12/16/24/32）。

### 5.5 布局栅格【实证 + 推断】

- 整体：**左固定栏（~240px）+ 右流动内容区**（`calc(100% - 240px)`，左 padding ~24px）。
- 资源广场：每行 **4 列**卡片瀑布流，自适应。
- 对话页：内容居中、垂直流式。

### 5.6 空状态 / 加载态【实证】

- **空状态/首屏**：欢迎语 + 多模态输入框 + "大家都在用"提示词推荐（不让用户面对空白）。

## 6. 交互范式总结

1. **对话即应用（Chat as Application）**【实证】：一个对话流里混排纯文本、结构化表单、可折叠卡片、进度条、交付物卡片，层次清晰——把复杂生成过程装进聊天范式。
2. **结构化参数 + AI 预填默认值**【实证】：表单字段全部预填，教师只需"确认/微调"，认知负担从"输入"转到"判断"。
3. **专家思考过程可视、可折叠**【实证】：把 AI 黑盒打开（课标研读/资源/逐页设计），增信任、默认折叠不打扰。
4. **Plan → Confirm → Execute + 2 个 HITL 门控**【实证】：规划后停下让教师确认参数，再执行，降低废稿率、给掌控感。
5. **分批生成 + 实时进度**【实证】：长任务拆 9 Batch、逐条进度，反等待焦虑。
6. **强互动页/普通页分级**【实证】：AI 主动标注哪些页"强互动"并给出交互规格，质量可控、预期透明。
7. **追问修复闭环**【实证】：生成不是终点，可追问→定向修复→新版卡片 + 修复摘要（"改了什么"透明）。
8. **提示词引导 + 社区热度**【实证】：高频提示词 + 热度数字 + 广场浏览量，营造从众与示范，降门槛。
9. **改编优先**【实证】：历史任务/示例大量"改编《…》/基于这份教案修改"，复用他人作品是一等公民。

## 7. 产品策略分析

**它在赌什么**
- 赌"**AI 生成可交互课件**"这一开环赛道：教师备课提效的刚需 + 一句话出专业课件的体验落差足够大。
- 赌"**把生成做到足够好就够了**"：用多阶段 workflow、专家思考、强互动页、追问修复堆出"成品感"。

**它强在哪**【实证】
- 生成链路完整且成熟：参数采集→专家思考→分批生成→交付→追问修复，闭环顺滑。
- 强互动页是真前端实现（Canvas/SVG/anime.js），非截图或套壳。
- 资源广场 + 改编机制形成内容飞轮（PGC 官方示范 + UGC 教师作品）。
- 交互细节克制专业（预填默认值、可折叠思考、进度反焦虑、缺资料告警）。

**它的边界在哪**【实证（缺失证据）+ 推断】
- **开环、止步于下载**：无"推送到课堂/发布作业/分享班级"动作入口。
- **不回收学生作答数据**：课件内 `postMessage` 上抛但宿主不消费，作答在 iframe 边界断流；无 SCORM/xAPI。
- **无学情绑定**：生成不感知班级历史表现，无法据薄弱点个性化。
- **生成与使用脱钩**：与 ClassIn 等教学平台无集成，"生成"和"上课/作业/学情"是两座孤岛。
- **资源孤岛**：修复后的新版课件存在于对话历史里，缺资源库级的协作/版本治理（推断）。

> 对「ClassIn AI 助教升级」的产品启示：飞象的**终点（下载一个 HTML）恰是 ClassIn 的起点**。差异化不在"比飞象生成得更快/更好看"，而在"**把生成的内容投进真实教学场景（预习/课堂/作业）并把作答数据收回来反哺生成**"——别人开环、我们闭环。详见技术篇 §10。
