## 1. 产品概述

**飞象老师 = 面向中学教师的 AI 互动课件生成工具，终点是下载一个可离线运行的 HTML 文件。**

- **核心价值主张**：教师填写结构化参数后，AI 自动完成课程标准研读、教学目标设计、逐页内容规划与分批代码生成，最终输出包含真实动画交互的 HTML 课件
- **核心优势**：
  - 生成流程完整：参数采集 → 专家思考 → 分批生成 → 交付物卡片，链路闭环
  - 强互动页质量高：热机循环（Canvas + anime.js）、电路连线（SVG + 拖拽逻辑）等交互使用真实前端技术实现，并非截图或 iframe 嵌套
  - 支持追问修复：生成后用户可追加反馈（如"修复公式渲染问题"），AI 对同一文件执行定向修复并产出新版交付物
- **核心缺陷**：
  - 生成完成即结束，无法一键推送到课堂、作业或班级群
  - 无学情数据绑定，课件被下载后 AI 侧完全失去可见性
  - 无 SCORM/xAPI 回收机制，学生作答数据无法归集
  - 与教学平台（ClassIn 等）完全脱钩，"生成"与"使用"是两个独立孤岛

### 3.3 专家思考卡片

专家思考以多个可折叠卡片形式呈现，每张卡片对应一个推理步骤。

**卡片类型与内容：**

| 卡片 | 图标 | 标题 | 内容形式 | 是否可折叠 |
|------|------|------|----------|-----------|
| 关键词提取 | 🔍 | 专家思考 — 关键词提取 | 标签数组，如 ["热学","电磁学","物理核心素养","内能","电路基础知识"] | 是 |
| 课程标准研读 | 📖 | 研读课程标准 | 结构化文本（学业质量标准多条 + 素养目标定位四维度） | 是（右上角"展开"按钮） |
| 教材内容检索 | 📚 | 教材内容检索 | 若无上传资料：橙色警告文字"当前未获取到有效单元内容解析内容，请提供对应的单元解析文本" | 是 |

| 目标与重难点 | 🔍 | 专家思考 — 目标与重难点刻 | 学习目标（4条）/ 学习重点（1条）/ 学习难点（1条） | 是 |
| 逐页内容设计 | 🔍 | 专家思考 — 逐页内容设计 | 表格（页码 / 内容类型 / 交互规格说明） | 是 |

**逐页交互设计示例（第6页 热机循环）：**
- **交互元素**：4个冲程按钮（吸气、压缩、做功、排气），中间为发动机汽缸示意图
- **过程反馈**：点击按钮后，汽缸内活塞随曲轴同步上下移动，进气门/排气门根据冲程状态开闭，燃料混合物或废气以粒子形式流动
- **结果反馈**：完成一个循环后，飞轮旋转一周，显示"完成一个工作循环，对外做功一次"字样
- **完成态**：点击"连续运行"，发动机自动循环，显示能量转化统计（化学能→内能→机械能）
- **技术**：Canvas 绘制汽缸结构 + anime.js 控制活塞与曲轴的同步平滑动画

**逐页交互设计示例（第11页 串并联电路连线）：**
- **交互元素**：左侧工具栏包含电池、开关、两个小灯泡；右侧为接线区
- **过程反馈**：拖动导线（SVG 动态绘制）连接元件触点，连接时触点高亮
- **结果反馈**：正确形成通路后点击开关，灯泡发光并显示"通路"；短路或断路则显示对应警告提示
- **完成态**：分别完成串联和并联连接，获得"电路工程师"勋章，并对比两种电路特点
- **技术**：SVG 动态绘图 + 自定义拖拽逻辑（mousedown/mousemove）+ 实时电路逻辑判定算法

### 3.7 用户追问与反馈机制

**追问卡片视觉：**
- 用户追加的反馈以绿色背景的按钮样式卡片呈现，与 AI 输出的白色区块形成色彩区分
- 示例："修复所有的公式没有正常渲染的问题"

**修复执行流程：**
1. 用户输入追问 → 卡片以绿色背景插入到对话流中
2. AI 输出单条进度行：`🔄 [文件名] 修复课件公式渲染问题`
3. 修复完成后进度行变为 `✓`
4. 产出全新版本的交付物卡片（与初版外观相同，为新版本文件）
5. AI 输出修复内容摘要

**修复内容摘要（公式渲染修复示例，共3项）：**
1. **规范定界符**：将所有错误的 `$` 符号替换为 MathJax3 标准行内定界符 `\( \)` 和块级定界符 `\[ \]`
2. **注入渲染配置**：显式注入 MathJax 配置脚本，确保公式在不同浏览器环境下稳定渲染
3. **公式排版优化**：
   - 热量计算：修复比热容单位 J/(kg·℃) 及公式 Q=cmΔt 的显示问题
   - 欧姆定律：强化主公式 I=U/R 的视觉权重，修复变形式和注意事项中的变量符号
   - 汇总表格：统一复习清单中所有物理量符号、单位及计算公式的 LaTeX 渲染

## 4. 交互设计亮点

### 4.1 结构化参数采集 + AI 预填默认值

表单中所有字段均由 AI 预先填写合理默认值，教师不需要从零输入，只需"确认"或"微调"。这大幅降低了使用门槛，将认知负担从"输入"转移到"判断"。

### 4.2 专家思考过程可见

AI 的推理过程（课标研读、资源检索、目标设计、逐页规划）以卡片形式透明呈现，并支持折叠。这一设计解决了 AI 黑盒问题：教师可以验证 AI 的依据是否正确，增加信任感，同时不强制阅读（默认折叠）。

### 4.3 分批生成 + 实时进度反馈

将 20 页课件拆分为 9 个 Batch 分批注入，每个 Batch 有独立进度行，实时从 `🔄` 变为 `✓`。这解决了长时间等待的焦虑感问题，用户能感知"任务在推进"而非面对转圈等待。

### 4.4 强互动页与普通页分级处理

AI 主动区分"强互动页"（4页）和"普通页"（16页），并为每种强互动页输出详细的交互规格（交互元素、过程反馈、结果反馈、完成态、技术方案），确保生成质量与技术选型透明可控。

### 4.5 追问修复闭环

生成完成后不是终点，用户可以继续追问触发定向修复。修复后产出全新版本卡片，并附详细的修复内容摘要，让用户知道"改了什么"。这是一个完整的"生成 → 反馈 → 修复 → 交付"闭环。

### 4.6 交付物卡片设计

卡片同时提供"缩略图预览"和"下载"两种操作路径，缩略图让用户在下载前能快速判断成品外观是否符合预期，降低下载决策成本。

### 4.7 对话中嵌入结构化内容

整个对话窗口混合了：纯文本对话、结构化表单、折叠卡片、进度指示器、交付物卡片等多种内容类型，且视觉层次清晰。这是典型的"对话即应用"（Chat as Application）设计范式。

## 5. 与 ClassIn 集成的差异化机会

### 5.1 飞象的边界与局限

| 限制点 | 具体表现 |
|--------|----------|
| 课件止步于下载 | 生成完成后，教师需手动下载 HTML 文件，再通过其他方式分发给学生或用于上课 |
| 无教学场景落地 | 没有"推送到课堂"、"发布为作业"、"分享到班级"等动作入口 |
| 无学生作答回收 | HTML 文件在本地运行，学生交互数据（拖拽轨迹、答题结果）无法上报任何平台 |
| 无学情绑定 | 课件生成不依赖也不感知学生学情数据，无法基于班级历史表现做个性化调整 |
| 无版本管理协作 | 修复后的新版本卡片独立存在于对话中，无法与其他教师共享或纳入资源库管理 |
| 无 SCORM/xAPI | 课件不符合任何 e-Learning 标准，无法被 LMS 追踪和记录 |

### 5.2 ClassIn 的差异化切入点

飞象的"终点"恰好是 ClassIn 的"起点"。ClassIn 可以在以下方向构建飞象不具备的能力：

**① 一键推送到 ClassIn 课堂**
- 在 AI 课件生成完成后，直接提供"推送到 ClassIn 课堂"按钮
- 教师无需下载再上传，课件直接进入 ClassIn 课堂白板或资源库

**② 学生作答数据回收**
- 将飞象生成的交互页（电路连线、拖拽分类等）封装为 ClassIn 可追踪的互动组件
- 学生在 ClassIn 课堂内完成交互，数据实时同步至教师端看板

**③ 学情驱动的课件生成**
- ClassIn 拥有班级历史作业、测验数据，可将"易错知识点分布"作为额外参数传入课件生成阶段
- 生成的课件在内容侧重上精准对应该班级的薄弱点，而非通用模板

**④ 课件资源库管理**
- 将生成的课件纳入 ClassIn 资源库，支持教师间共享、复用、版本迭代
- 解决飞象"只存在于对话历史中"的资源孤岛问题

**⑤ 作业与课后练习闭环**
- 将课件中的练习页（第7-10页、第15-20页）提取为 ClassIn 作业题目
- 学生在 ClassIn 完成练习，AI 自动批改并更新学情画像，驱动下一节课的课件生成

**⑥ 家长/学生端可见性**
- 教师在 ClassIn 分发课件后，学生在 ClassIn App 内直接访问，无需下载 HTML 文件
- 家长可在家长端看到孩子的课件学习进度和互动参与情况

### 5.3 定位建议

| 维度 | 飞象老师 | ClassIn + AI（差异化定位） |
|------|----------|--------------------------|
| 产品定位 | AI 课件生成工具（内容侧） | AI 教学场景平台（场景侧） |
| 核心动作 | 生成 → 下载 | 生成 → 推送 → 互动 → 数据回收 → 学情更新 |
| 数据归属 | 课件文件（离线，无数据） | 课堂 + 学生 + 学情（在线，全链路数据） |
| 商业模式 | SaaS 订阅（生成次数/容量） | 教学平台生态（课堂 + 作业 + 学情 + 家校） |
| 核心壁垒 | AI 生成质量 + 使用便捷性 | 教学数据 + 学情闭环 + ClassIn 生态深度集成 |
