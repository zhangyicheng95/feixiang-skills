## 0. TL;DR（研发视角一句话）

> 飞象 = **React SPA 壳（Hash 路由 + featureId）** + **多阶段 agent/workflow 编排（HTTP 短轮询拉状态，非 SSE/WS）** + **LLM 从零生成自包含 HTML 课件** + **工程化外壳（iframe srcdoc 宿主 + 埋点 + 共享库 CDN + 发布后处理噪声）**。
> 关键技术判断：**只有"外壳"被工程化，"内层课件内容"是 LLM 每次现写、零复用、无 schema** —— 这决定了它能"演示互动"但**无法结构化采集作答数据**。

## 2. Agent / Workflow 编排全流程

实测案例：**上传 71.5KB 九年级物理复习 Word → 输出 20 页课件**，全程 8+ 阶段、2 个用户确认门控（HITL）、9 个 Batch 串行生成。【实证，完整交互日志】

```text
①意图理解&教学设计 → ②大纲/分页规划(outline) → [用户确认门控①]
   → ③专家思考(~2000字) → ④逐页大纲设计 → [用户确认门控②]
   → ⑤图片生成(文生图) → ⑥复杂度评估(强互动 vs 普通)
   → ⑦分批串行生成(9 Batch + 注入骨架) → ⑧素材回填 → ⑨互动/状态装配
   → ⑩后处理(噪声/水印) → ⑪外层壳封装(iframe + 埋点) → 交付物卡片
   → [追问] → ⑫增量定向修复 → 新版交付物卡片
```

**任务规划 5 步（专家思考阶段，实证）**：课标研读 → 教材梳理 → 学情分析 → 目标拟定 → 大纲设计。
**信息确认（HITL①）字段**：年级刻 / 册次刻 / 教材版本刻（人教/北师大/苏科…）/ 教学重点刻（多选）/ 课时数刻 / 特殊要求。
**专家思考可折叠卡片**：🔍关键词 → 📖课标研读（学业质量标准 + 素养目标四维）→ 📚教材检索（缺资料橙色告警）→ 📖精选资源（外链 PDF）→ 🔍目标与重难点 → 🔍逐页设计表。
**产物**：交付物卡片（HTML 图标 + 缩略图 + 下载）。

### 2.1 两种生成范式并存（置顶任务命名实证）

| 范式 | 暴露任务名 | 适用 |
|---|---|---|
| **Agent 自主** | 教学动画-Agent自主规划、练习数据回收-Agent自主调用、互动游戏-Agent自主调用 | 灵活任务，Agent 自主规划/调用工具 |
| **工作流** | 文档生成-工作流生成、多页互动课件-工作流生成 | 复杂可控的多页/文档生产 |

### 2.2 可提炼的编排范式特征【实证(日志)+推断】

1. **复杂度感知路由**：简单单页课件走"单次 LLM 生成"快通道；复杂多页课件走"完整 workflow"。两套策略并存。
2. **Plan → Confirm → Execute + HITL 门控**：先规划、用户确认参数、再执行。
3. **交互剧本中间层**：写代码前先把"教学需求"翻成"技术方案"（如"Canvas 绘汽缸 + anime.js 控活塞"），强互动页额外编"交互剧本"。
4. **分批串行 + 骨架组装**：强互动页单独占 1 Batch，普通页 2–4 页合并，注入统一骨架。
5. **增量定向修复**：按反馈只改目标段，不全量重来。

### 2.3 能力 = Skill（Prompt 约束规则，非代码组件库）【推断，来源 A】

一份 Skill 含：角色定义、CDN 库声明 + 触发条件、CSS 变量强制、交互模式指南、壳框架兼容规则（白名单类名）、单/多页输出格式约束、**反噪音约束**（禁止 AI 幻觉 build marker/GA4/CMS）、学科正确性约束。
> 方向相反的两件事：**生成阶段反噪音**（Skill 禁幻觉元数据）vs **发布阶段加噪音**（后处理注入垃圾，见 §6.4）。

确证的交互模式地图：单选/判断、分步揭示、3D 拖拽旋转、anime 动态演示、原生 DnD 建模、DnD 归类、Canvas 白板、点击填空、双人竞技、关卡闯关、图片热区、视频嵌入。

## 3. 流式机制（★关键实证）

> **agent runner 用 HTTP 高频短轮询拉执行状态，不是 SSE / WebSocket。**

| 项 | 结论 | 证据 |
|---|---|---|
| 机制 | **HTTP 短轮询** | 【实证，勘察 E】 |
| 端点 | `/musk-ai-studio/api/agent-chat/executing-status` | 【实证】 |
| 频率 | 约 **80–150ms / 次** | 【实证】 |
| 非 | 非 SSE、非 WebSocket | 【实证】 |
| 文字呈现 | **整块出现**，非逐字流式 | 【实证】 |

> 设计动机（推断）：批量生成耗时长、阶段多，短轮询比长连接更稳、更易在多阶段 workflow 上做状态机轮询；代价是请求数高、非逐字流式体验。
> ⚠️ 与续段遍历报告里"SSE streaming 响应"的写法冲突——**以本节短轮询实证为准**，续段的 SSE 是行业惯例推断。

## 5. 课件产物运行时与内容数据结构

### 5.1 两种交付形态【实证】

- **单页课件**：一份完整 HTML 塞进 `iframe srcdoc`，自包含。
- **多页课件**：`page-shared`（设计令牌 + 共享库）+ 20 个 `<template class="page-data" data-id data-name>` 分页 + 壳框架翻页脚本（手势/键盘/缩略图/演示模式）。

### 5.2 页骨架三件套【实证，courseware 4561–5485 行】

- 分片容器 `<template class="page-data" data-id data-name>`：每页一个，`data-id` 连续，`data-name` 带教研语义（封面/复习框架/热机四冲程/串并联电路连线…）。
- 页容器 `.page-container`（18/20 页用；封面/转场/结课为全屏特例）。
- 组件基线类：`.page-title`（32px/700/左 6px 主色竖线）、`.card`（圆角 12 + 阴影）、`.btn`/`.btn-primary`。
- **栅格几乎全靠行内 style 写 flex/grid**，未抽象栅格类 → outline 只约束"页级骨架"，不约束"块级布局"。

### 5.3 壳框架事件拦截机制【实证】

壳框架拦截页面空白处点击用于翻页，用**约定 CSS 类名白名单**自动放行可交互元素（`.tool-btn / .drag-item / .drop-zone / .option-btn / .input-box / canvas / video …`），并用 `MutationObserver` 监听动态元素。
→ AI 生成课件只要**遵守类名约定**即可被壳框架识别。这是"约定优于配置"的轻量运行时，**不是 schema 驱动的渲染引擎**。

### 5.4 内容数据结构 = 无 schema / 无 DSL（★关键）【实证，来源 B】

内容就是 HTML 代码本身，**没有"题目对象/选项数组/答案字段"这一层**：

- 单选题：`checkQ7(this, true/false)` —— **正确答案以布尔字面量硬编码在 `onclick`**，题干/选项/解析全是 DOM 文本。
- 分步揭示：`showStep()` 计数器自增。
- 动态演示：`runStroke(index)` 数据数组驱动绝对定位 DIV + anime（**非真 `<canvas>`**：搜 `getContext` 0 命中；电路页 `<svg>` 留空未用）。
- 拖拽建模：`testCircuit()` 仅查"元件是否齐全"，**不验真实连线拓扑**（AI 自述"简化版…不含复杂实时 SVG 拓扑"）。
- 拖拽归类：`handleDrop()` 靠 `data-type` 字符串相等匹配。
- 命名约定：函数=动词+对象（check/show/run/play/test），id=语义-序号（step-1/zone-series）。
- **零组件抽象**：同名 `saveState()`/`state` 在多页各写一份，靠 `<template>` 隔离作用域 → "每页一份拷贝"，非模块化。

→ 后果：**表达力极强，但不可结构化采集、不可版本化演进、不可标准化质检。**

### 5.5 状态协议 = postMessage（有名无实）【实证】

课件内 `window.parent.postMessage({type:'saveState'|'restoreState', state}, '*')` 上抛，但 **state 形状逐页自定**（`{mode}`/`{currentStroke}`/`{items}`/`{correctCount}`），无统一 schema，无法跨页聚合；恢复能力参差。**且 parent 根本不消费 saveState** → 作答数据在 iframe 边界断流。

### 5.6 素材 / 资源管线【实证】

- 配图：文生图 → 对象存储（22 位随机 ID）→ 回填 `<img src>`，与课件 HTML 同 `workflow/file/` 目录。20 页仅 4 张图 → "**能代码画就代码画，画不出的真实照片才文生图**"的节流规范。
- 共享库走 CDN 白名单（`metis-misc/*.js`），课件只引用不内联；多样本复用同一组库文件。
- **CSP**：仅约束 `img-src`/`media-src`（自有 fbcontent + 阿里云 OSS + GCS），**不约束 script/style** → 噪声里的假外链不会真请求。

### 5.7 SCORM / iframe / 标准【实证】

- iframe：外层壳 `iframe srcdoc`（HTML 实体编码）+ `__SERVER_DATA__` 注入 4 个 watermark 变量 + `file://` 重定向。
- **SCORM / xAPI：完全没有**（资料零证据）。课件不符合任何 e-Learning 标准，无法被 LMS 追踪。
- 画布标称 960×540（固定 px），自适应靠外层壳等比缩放（移动端 375 rem 基准）；**内层固定 px，无 rem/vw/clamp/@media**。

### 5.8 前端库栈（课件层）【实证】

| 库 | 用途 |
|---|---|
| MathJax | 公式渲染（`\(\)` 行内 / `\[\]` 块级） |
| anime.js | 声明式属性动画 + `timeline()` + `update()` 钩子 |
| Three.js + OrbitControls + GSAP | 3D 课件渲染/动画 |
| Noto Sans SC（Google Fonts） | 字体（抓取时被抹为 `[unsafe-url-removed]`） |
| 图标 | 全用 emoji（🔥⚡🔋💡），零图标库 |
| `metis-misc/*.js`（3 随机名） | 共享工具库（推断为 anime/MathJax/自研） |

## 6. 数据闭环能力（能 / 不能回收什么）

| 能力 | 飞象现状 | 证据 |
|---|---|---|
| 页面停留时长 | ✅ 能（埋点 `use_time`，多事件 + 100ms 防抖累加） | 【实证】 |
| 品牌点击（logo / "立即体验"导流官网） | ✅ 能 | 【实证】 |
| 学生作答（题/答案/对错/用时/尝试次数） | ❌ 不能（saveState 上抛但 parent 不消费，iframe 边界断流） | 【实证】 |
| SCORM / xAPI 标准回收 | ❌ 无 | 【实证（缺失）】 |
| 学情沉淀 / 数据反哺生成 | ❌ 无（知识单向：知识→生成，无回流） | 【推断】 |
| 与 LMS / 班级 / 课堂集成 | ❌ 无（生成与使用两孤岛） | 【实证】 |

> **结论：飞象是"开环生成"。** 埋点投入全压在"停留时长"一个指标，恰恰反衬作答数据完全未采。这是飞象与"需闭环采集"产品的根本分水岭。

## 7. 跨样本结论："软规范 vs 硬外壳"（★核心技术判断）

> 用 Grep 抽样 4 个课件样本（courseware / s2 / s3 / s4 的 `.inner.html`）交叉验证。【实证 D】

### 7.1 被工程化的"硬资产"（跨样本一致）

- **共享前端库**：`metis-misc/zgLDUdmazTYc0B4K6Cor.js`、`zZZY40t7WJC7UdQCPACm.js`、`blER0Bn7vsa2JER9IEssf8.js` 等同一组文件跨样本复用 → 库层是固定白名单。
- **外层壳 + iframe 宿主 + 埋点**：结构跨样本一致。
- **后处理噪声层**：`WCAG 2.1 AA`、`Webpack 5.88.2`、`Marker-xxxx`（`left:-9999px` 离屏隐写水印）、`data-measurement-id`(ga4)、`feature-flags`、统一 CSP 等假元数据成体系注入。

### 7.2 靠约定的"软规范"（跨样本各异）

- **设计令牌值逐课件不同**：`--primary` 在四样本分别是
  `#3B82F6`(courseware) / `#2563EB`(s4) / `#01553D`(s3) / `#D7F324`(s2)，`--accent` 亦各异。
  **令牌名一致、取值由 LLM 现场决定**。`page-shared` 的 `:root` 仅定义 7 变量（`--primary/--secondary/--accent/--bg/--text/--success/--error`），但状态反馈色大量写死 hex（正确 `#dcfce7/#22c55e`、错误 `#fee2e2/#ef4444`），未复用令牌。
- **内层 DOM/CSS/JS 零代码复用**：每份课件各写各的，互动函数手写，命名约定相同但实现不共享。

### 7.3 核心判断

> **内嵌"设计系统"是软规范**（靠 prompt/示例约束，非硬资产）：**只有外壳被工程化**（埋点、iframe 宿主、共享库、噪声/水印），**内层内容由 LLM 生成、风格一致性靠约定维持**。所以跨样本能看到"名字一样、值不一样、代码不复用"。飞象的"设计系统"不是一套可复用组件/主题代码，而是一组写进 Skill 的命名与令牌约定。

## 10. 对「ClassIn AI 助教升级」的技术启示

> 我方目标：让 ClassIn AI 助教从 RAG 问答升级为"替教师完成教学动作"的 agentic 入口，第一条链路是「互动内容生成 · 多场景交付 · SCORM 数据回收 · 反哺生成」。

### 10.1 可直接借鉴（飞象已验证可行）

| 借鉴点 | 飞象做法 | 对我方意义 |
|---|---|---|
| 多阶段 agent workflow | 8+ 阶段 + 2 HITL，非单次调用 | 印证"生成=agent/workflow"方向，可参考阶段划分 |
| 复杂度感知路由 | 简单走单次、复杂走 workflow | 控成本/延迟：简单题型快通道、复杂内容走全流程 |
| Plan→Confirm→Execute + HITL | 规划后让老师确认参数 | 增掌控感、降废稿率 |
| 交互剧本中间层 | 写码前先把需求翻成技术方案 | 规划→生成间加方案设计层，提可控性 |
| 教育知识引擎做生成上下文 | 课标/教材/学情注入 | 印证"知识库是生成认知底座" |
| CDN 库白名单 + 反噪音约束 | 限定库、禁幻觉生产代码 | 约束输出范围 + 做净化 |
| 分批生成 + 骨架组装 | 长内容分 Batch 注入骨架 | 长篇/多题内容的稳妥工程方案 |
| 增量修复 | 反馈后只改目标段 | 对应"教师可编辑修正"，避免全量重生成 |

### 10.2 必须做得不一样（差异化护城河）

| 维度 | 飞象（参照系） | 我方必须不同 | 为什么 |
|---|---|---|---|
| **闭环 vs 开环** ★ | 只埋停留时长，无作答回收 | **必须闭环**：SCORM/xAPI 采集 → 学情沉淀 → 反哺生成 | ClassIn 护城河、项目核心价值 |
| **内容结构** ★ | 无 schema，整页 HTML，零复用 | **必须 schema/DSL 驱动**：题型组件库 + 内容运行时 player | 只有结构化内容能标准化采集作答；整页 HTML 路线天然采不到 |
| **运行时** | 翻页壳框架（约定类名放行点击） | **schema 驱动渲染引擎 + 行为埋点**，与 SCORM 运行时衔接 | 要采"哪题/答案/对错/用时"，需 player 统一接管交互与采集 |
| **生成产物** | 独立 HTML，浏览即止 | **可投放进 ClassIn 预习/课堂/作业/表单**，对接 LMS 活动体系 | 飞象不与 LMS/教室/班级集成，我方原生进场景 |
| **知识流向** | 单向：知识→生成 | **双向**：知识库供给生成 + 作答回流更新学情/反哺 | 飞象无"记忆系统"，我方闭合数据回路 |
| **质量保障** | 无后处理校验，噪音原样保留 | **校验/重生成 + 人工可修正** | 教学内容与作答数据正确性要求高 |

### 10.3 三条给 PM / 研发的关键判断

1. **架构分水岭在"内容结构"**：飞象证明"整页 HTML 一次性生成"在**纯展示型课件**上可行且表达力强，但**无法支撑结构化作答采集**。我方要做数据闭环就**不能走无 schema 路线**，须在 Phase 0 定下"内容 schema/DSL + 题型组件 + 运行时 player"地基。
2. **workflow 可"抄作业"，闭环要自己造**：生成侧的多阶段编排、复杂度路由、HITL、交互剧本、增量修复都被验证过，可大胆参考；但 SCORM/xAPI 采集→学情沉淀→反哺这条**数据回路飞象根本没有**，是真正拉开差距处。
3. **差异化叙事成立**：飞象代表"AI 生成互动课件"开环赛道的高水位（多模态、20 页复杂课件、专家级教研逻辑），但止步于"生成+交付独立 HTML"。**ClassIn 手里的在线教室、LMS、SCORM、班级数据是飞象拿不到的闭环要素**——价值在"把生成内容投进真实场景并把数据收回来反哺"。
