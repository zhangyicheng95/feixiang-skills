# math-design 标准教具样式

> 仅数学场景可读取。非数学场景禁止读取本文件。

本文件约束数学标准教具的固定视觉样式。当前已定义：标准二维坐标轴。

## 读取条件

当页面实际绘制以下内容时，必须读取本文件并执行对应章节：

- 平面直角坐标系
- x/y 坐标轴
- 函数图像坐标轴
- 坐标平面中的点、线、图形、变换演示

只要出现 x 轴和 y 轴，就必须使用本文件的“标准二维坐标轴组件”作为基础样式。

## 标准二维坐标轴组件

### 硬约束

- 坐标轴、箭头、`x` 字母、`y` 字母必须位于同一个 SVG 根节点中。
- 禁止把 `x` / `y` 做成脱离坐标轴 SVG 的独立 HTML 文本、绝对定位元素或单独 SVG 浮层。
- 根 SVG 的 `viewBox` 固定为 `0 0 308.0001 308.0001`。
- 标准色固定为 `#251F20` / `rgb(37, 31, 32)`，除非用户明确要求换色；换色时所有轴线、箭头、x/y 字母必须同步换色。
- 轴线视觉粗细固定为 `1.5px`，通过原始路径厚度实现，不改成浏览器默认线条。
- x/y 字母使用标准路径 glyph，不使用系统字体替代。
- 用户要求放大或缩小时，只能缩放整个坐标轴根 SVG，不得分别移动或缩放轴线、箭头、x/y 字母。

### 固定几何关系

标准画布：

```txt
canvasSize = 308.0001
axisCenter = 154.082
axisLength = 301.249
axisColor = #251F20
```

固定位置：

```txt
横轴线：translate(0, 154.082)
横轴箭头：translate(295.819, 149.65)
纵轴线：matrix(0 -1 1 0 154.082 308)
纵轴箭头：matrix(0 -1 1 0 149.65 12.1808)
y 字母：translate(133.989, 0.870118)
x 字母：translate(295.819, 167.051)
```

抽象关系：

```txt
y 字母固定在纵轴箭头左侧，并随纵轴整体缩放。
x 字母固定在横轴箭头右下侧，并随横轴整体缩放。
```

等价公式：

```txt
yLabel.x = axisCenter - 20.093
yLabel.y = 0.870118

xLabel.x = canvasSize - 12.1811
xLabel.y = axisCenter + 12.969
```

这些数值属于标准样式的一部分。生成代码时不得重新估算。

### 缩放规则

如果用户要求调整坐标轴大小：

- 保持 `viewBox="0 0 308.0001 308.0001"` 不变。
- 保持内部所有 `path`、`transform`、`width`、`height` 比例关系不变。
- 只调整根 SVG 的 CSS 尺寸，例如 `width` / `height` / `max-width` / `max-height`，或调整外层容器的等比 `transform: scale(...)`。
- 缩放必须等比，禁止只改宽度或只改高度。
- x/y 字母位置必须随根 SVG 一起缩放，禁止缩放后再单独用 `left/top/translate` 修正。

推荐包装：

```html
<div class="standard-axis-wrap">
  <!-- 粘贴下方 standard-axis SVG -->
</div>
```

```css
.standard-axis-wrap {
  width: min(42vmin, 308px);
  aspect-ratio: 1 / 1;
  display: flex;
  align-items: center;
  justify-content: center;
}

.standard-axis {
  width: 100%;
  height: 100%;
  display: block;
}
```

更大或更小的坐标轴只改 `.standard-axis-wrap` 的 `width`。不要改内部 `path` 和 `transform`。

## 数据点、函数曲线与刻度

标准坐标轴本身只约束轴线、箭头和 x/y 字母。若题目需要点、函数曲线、三角形、辅助线或刻度：

- 必须添加在同一个 SVG 内，作为附加数据图层。
- 附加数据图层不得覆盖、移动或重写 6 个标准路径；若需要控制层级，应让 x/y glyph 保持在最上层。
- 数据图层必须使用同一套坐标换算，原点对齐 `axisCenter = 154.082`。
- 默认不画背景方格；只有用户明确要求“方格线 / 网格线 / 坐标网格 / 坐标纸 / 方便读点”等，才按 `workflow.md` 的方格规则追加网格层。

推荐坐标换算：

```js
const AXIS_SIZE = 308.0001;
const AXIS_CENTER = 154.082;
const UNIT = 24;

function toScreenX(x) {
  return AXIS_CENTER + x * UNIT;
}

function toScreenY(y) {
  return AXIS_CENTER - y * UNIT;
}
```

如果用户要求更大的坐标显示范围，优先调整 `UNIT` 或数据图层范围；不要拉长标准轴线内部路径。若确实需要更大视觉尺寸，缩放整个 SVG。

## 禁止写法

- 禁止把 x/y 写成 `<text>x</text>` / `<text>y</text>` 后用 CSS 绝对定位。
- 禁止将 x/y 字母拆成两个独立 SVG，再用 `left/top` 放到轴线旁。
- 禁止只缩放横轴或只缩放纵轴。
- 禁止用户要求“放大坐标轴”时只改轴线长度而不改 x/y 字母位置。
- 禁止用户要求“缩小坐标轴”时压缩轴线但保留原字号 x/y 字母。
- 禁止用默认浏览器线条、默认箭头 marker 或系统斜体字母替代标准路径。

## 自检

生成或修改含二维坐标轴的 HTML 后，terminate 前必须确认：

- 根 SVG `viewBox` 仍为 `0 0 308.0001 308.0001`。
- `.standard-axis` 内存在 6 个标准 `path`：横轴、横箭头、纵轴、纵箭头、y glyph、x glyph。
- y glyph 的 transform 仍为 `matrix(1 0 0 1 133.989 0.870118)`。
- x glyph 的 transform 仍为 `matrix(1 0 0 1 295.819 167.051)`。
- 坐标轴缩放只发生在根 SVG 或外层 wrapper 上。
- x/y 字母随坐标轴一起放大或缩小，没有单独漂移。
