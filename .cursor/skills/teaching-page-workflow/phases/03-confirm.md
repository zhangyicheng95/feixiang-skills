# Phase 3：大纲确认

**输入**：`outline.yaml`  
**输出**：定稿 `outline.yaml`（可原地更新）

## 步骤

1. 用简短列表向用户展示大纲（页序 / 关卡 / 互动点）
2. 询问是否需要调整（Cursor 对话确认即可，无需飞象 `ask_user` 工具）
3. 若有修改 → 更新 `outline.yaml` → 再展示一次
4. 用户无异议 → 在 yaml 顶部加 `confirmed: true` 与 `confirmed_at` 日期

## 展示格式（示例）

```
拟定大纲（4 页互动课件）：
1. 封面
2. 概念讲解 — 中和反应定义
3. 滴定模拟 — 滑块调碱量，pH 变化
4. 课堂测验 — 4 题选择，提交上报 SCORM
```

游戏示例：

```
拟定大纲（课程游戏 · 2 关）：
关 1 入门 — 3 题选择，即时反馈
关 2 进阶 — 4 题选择，计分结算 → SCORM
```

## 可跳过

用户首条消息已给**逐页明细**且明确「按此生成」→ 可标 `confirmed: true` 并注明 `skip_reason: 用户已逐页指定`。
